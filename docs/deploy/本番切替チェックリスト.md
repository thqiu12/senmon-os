# 本番切替 チェックリスト（新サーバーへのクリーン構築）

前提：**本番は別の新VPS**を立てる。現ステージング（stg.system.chuo-seminar.ac.jp）の
出願データは**全てテスト**なので、**データ移行は不要**＝ゼロから構築する。

> 基本構築は [DEPLOY.md](./DEPLOY.md)、オフサイトバックアップは [OFFSITE-BACKUP.md](./OFFSITE-BACKUP.md) を参照。
> 本書はそれらを呼び出す「本番用の親手順」。本番特有の注意点と、今回判明した罠を明記する。

---

- [ ] DNS管理画面にアクセスできること（Aレコード設定用）
- [ ] GitHub の clone 手段（PAT or deploy key）

## 1. 基本構築（DEPLOY.md の 1〜3）

  - [ ] **デモデータは入れない**（本番なので。`prisma/seed.ts` の管理者のみ）
- [ ] `03-nginx-setup.sh`

## 2. 本番用 .env（ステージングと使い回さない）⚠️重要
`/srv/senmon/app/.env` を本番用に作成。**シークレットは新規生成**：
```bash
# セッション/CSRF シークレットは本番用に新規生成（stgの値を流用しない）
openssl rand -hex 32   # SESSION_SECRET
openssl rand -hex 32   # CSRF_SECRET
```
- [ ] `SESSION_SECRET` / `CSRF_SECRET` … 上記で新規生成した値
- [ ] `RESEND_API_KEY` … **本番用に新規発行**（stg用キーと分ける）
- [ ] `RESEND_FROM` … 例 `"専門学校 入学係 <no-reply@chuo-seminar.ac.jp>"`
- [ ] `NEXT_PUBLIC_BASE_URL` … 本番ドメイン
- [ ] `UPLOAD_DIR="private/uploads"`（相対のまま）
- [ ] `SEED_ADMIN_USERNAME` / `SEED_ADMIN_PASSWORD`（初回のみ。**初回ログイン後に必ず変更**）

> ⚠️ **DBの実パスの罠**：`DATABASE_URL="file:./prisma/data.db"` は Prisma が schema.prisma
> 基準で解決するため、SQLite実体は **`/srv/senmon/app/prisma/prisma/data.db`**（prisma二重）。
> uploads の実体は **`/srv/senmon/app/private/uploads`**。バックアップ系スクリプトは
> 既に両方を自動検出するよう修正済み（offsite-backup.sh / backup-daily.sh / auto-deploy.sh）。

## 4. セキュリティ・ハードニング（今回の手順を再実施）⚠️本番こそ必須
- [ ] **SSH鍵認証のみ**化（Macから `ssh-copy-id` → 鍵ログイン確認 → パスワード認証オフ）
  ```bash
  sudo tee /etc/ssh/sshd_config.d/00-hardening.conf >/dev/null <<'EOF'
  PasswordAuthentication no
  KbdInteractiveAuthentication no
  PermitRootLogin prohibit-password
  EOF
  sudo sshd -t && sudo sshd -T | grep -i passwordauthentication   # → no を確認
  sudo systemctl reload ssh
  ```
- [ ] **ufw**（SSH許可してから有効化）。**3000は許可しない**（アプリ直アクセス遮断）
  ```bash
  sudo ufw default deny incoming; sudo ufw default allow outgoing
  sudo ufw allow 22/tcp; sudo ufw allow 80/tcp; sudo ufw allow 443/tcp; sudo ufw allow 8443/tcp
  sudo ufw --force enable
  ```
  > ⚠️ Next.jsは `*:3000` で待受するため、ufwで3000を塞ぐことが**外部直アクセス遮断の要**。
- [ ] **fail2ban** + **unattended-upgrades** 有効化
- [ ] 確認：Macから `curl -m5 http://本番IP:3000` がタイムアウト（exit≠0）＝3000遮断OK

## 5. オフサイト暗号化バックアップ（OFFSITE-BACKUP.md）

## 本番特有の「忘れがちポイント」まとめ
1. **シークレットは全部新規**（SESSION/CSRF/Resendキー/R2トークン/バックアップパスフレーズ）。stg流用は厳禁。
2. **初期管理者パスワードを即変更**。
3. **バックアップパスフレーズはサーバー外保管**（無いと災害時に復号不能）。
4. **3000をufwで塞ぐ**（塞がないと平文HTTPでアプリが外部露出）。
5. DB実体は `prisma/prisma/data.db`、uploadsは `app/private/uploads`（スクリプトは自動検出済み）。
