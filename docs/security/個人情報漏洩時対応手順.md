# 個人情報 漏洩時 対応手順書（インシデント・レスポンス）

> ⚠️ 本書は実務テンプレートです。最終的な判断・運用は弁護士／行政書士等の専門家に確認してください。
> 個人情報保護法（APPI, 2022年改正）に基づく「個人情報取扱事業者」の義務を前提に作成。

対象システム：**専門学校 入学出願システム（senmon）**
扱う主な個人情報：氏名・フリガナ・生年月日・性別・国籍・在留資格・連絡先（電話/メール/住所）・出身校・志望校・証明写真・在留カード/パスポート等の提出書類。

## 1. 発覚〜初動（最初の1時間）— まず止血

1. **発見者は直ちに責任者へ報告**（時刻・状況をメモ：いつ/何が/どこで気づいたか）。
2. **被害拡大の停止（封じ込め）**：
   - 不正アクセスの疑い → 該当アカウントの無効化、`pm2 restart`、必要なら一時停止。
     ```bash
     # 全管理セッションを即時失効（tokenVersion を全件 +1）
     sqlite3 /srv/senmon/app/prisma/prisma/data.db "UPDATE AdminUser SET tokenVersion = tokenVersion + 1;"
     ```
   - 漏れたアカウント/キーがあれば**即ローテーション**（管理者パスワード、Resend/R2 トークン、SSH鍵）。
   - 外部公開の誤り → nginx/ufw で遮断。
3. **証拠保全**：消す前にログを退避（`/var/log/nginx/`, `/srv/senmon/backup/*.log`, `auth.log`）。
   ```bash
   sudo tar -czf /tmp/incident-$(date +%Y%m%d-%H%M).tar.gz /var/log/nginx /var/log/auth.log /srv/senmon/backup/*.log 2>/dev/null
   ```
4. この時点では**むやみに再構築・初期化しない**（原因・範囲が分からなくなる）。

## 2. 事実確認（数時間〜1日）

- **何が漏れたか**：対象テーブル/ファイル、項目（要配慮情報・書類画像の有無）。
- **何件・誰の分か**：影響を受けた本人の概数と範囲。
- **原因**：不正アクセス／設定ミス／誤送信／紛失 等。
- **悪意の有無**：外部攻撃（＝「不正の目的」あり）か、内部の過失か。

> 在留資格・国籍は機微な情報。証明写真や在留カード画像の流出は本人への影響が大きいので慎重に評価。

## 3. 報告義務の判定（重要）

APPI では、次の**いずれかに該当する漏洩等**は **個人情報保護委員会への報告＋本人への通知が義務**：

1. 要配慮個人情報が含まれる
2. 不正利用で財産的被害のおそれ（クレカ番号等）
3. **不正の目的による行為**（＝外部からのハッキング・持ち出し等）
4. **1,000人超**の本人に係る

→ **本システムでの現実的な漏洩（外部侵入 or 大量流出）は、ほぼ「報告義務あり」**と考えて動く。迷ったら報告する前提で。

### 報告のタイムライン（PPC）
- **速報**：把握後**速やかに**（概ね3〜5日以内目安）に分かる範囲で。
- **確報**：原則**30日以内**（不正の目的による場合は**60日以内**）に最終報告。
- **本人通知**：状況に応じ**速やかに**。連絡が困難なら代替措置（公表等）。

## 4. 本人への通知（テンプレ）

> 件名：【お詫び】個人情報に関するお知らせ
>
> ［学校法人名／学校名］です。このたび、当校の出願システムにおいて、お客様の個人情報が外部に漏洩した可能性があることが判明しました。心よりお詫び申し上げます。
> ・漏洩した可能性のある項目：［氏名・連絡先・…］
> ・経緯：［　　　　　］
> ・現在の対応：［封じ込め済み／調査中／再発防止策　等］
> ・お願い：心当たりのない連絡にご注意ください。ご不明点は下記窓口へ。
> 連絡窓口：［電話／メール／受付時間］

## 5. 復旧・再発防止

- 原因に応じた恒久対策（脆弱性修正・権限見直し・キー管理改善）。
- 必要なら**オフサイトバックアップから復元**（`scripts/deploy/offsite-restore.sh`、要パスフレーズ）。
- 対応の**記録を残す**（時系列・判断・連絡履歴）＝説明責任のため。

## 付録：このシステムの緊急操作早見表
- 全管理者セッション失効：`UPDATE AdminUser SET tokenVersion = tokenVersion + 1;`（上記）
- 特定管理者の無効化：`UPDATE AdminUser SET isActive = 0 WHERE username='xxx';`
- アプリ緊急停止：`pm2 stop senmon-nyuugaku`
- 外部遮断（例：443を一時閉鎖）：`sudo ufw deny 443/tcp`（復旧時 `sudo ufw allow 443/tcp`）
- Resendキー無効化：Resendダッシュボード → API Keys → Revoke
- R2トークン無効化：Cloudflare → R2 → Manage R2 API Tokens → 該当を削除
