# 本番設定チェックリスト（senmon / Compass）

コードはあるが**環境変数・サーバ設定で有効化が必要**な項目。`/srv/senmon/.env`（本番）に設定し、`pm2 restart` で反映する。
SSH/.env はオーナーのみが操作（運用ルール）。

## 1. メール送信 `RESEND_API_KEY`（最優先・サイレント障害になりうる）

未設定だと**出願番号の確認メール／管理者通知が一切飛ばない**。出願自体は成功する（エラーは握り潰して続行）ため、
気づかないまま「学生に出願番号が届いていない」状態になりうる。

```bash
# 確認（本番サーバ上）
grep RESEND_API_KEY /srv/senmon/.env   # 空 or 無ければ未設定

# 設定 → 反映
#   RESEND_API_KEY=re_xxxxx を .env に追記
pm2 restart senmon

# 動作確認: テスト出願を1件行い、確認メールが届くこと（+ 管理者通知が届くこと）
```

## 2. AIアシスト `ANTHROPIC_API_KEY`

未設定だと AI 機能（書類チェック／面接講評・合否提案／メール下書き）が**すべて no-op**。
キーを入れた瞬間に有効化される（コードは導入済み）。コストは利用量見合い。

```bash
grep ANTHROPIC_API_KEY /srv/senmon/.env
#   ANTHROPIC_API_KEY=sk-ant-xxxxx を .env に追記 → pm2 restart senmon
# 確認: 申請詳細の「書類チェック」カードが結果を返すこと
```

## 3. 監視 Sentry `SENTRY_DSN` / `NEXT_PUBLIC_SENTRY_DSN`

未設定だと監視が**未稼働**（本番エラーに気づけない）。サーバ側=`SENTRY_DSN`、ブラウザ側=`NEXT_PUBLIC_SENTRY_DSN`。
ソースマップ送信をするなら `SENTRY_ORG` / `SENTRY_PROJECT` / `SENTRY_AUTH_TOKEN` も（任意・ビルド時）。

```bash
#   SENTRY_DSN=https://....ingest.sentry.io/...
#   NEXT_PUBLIC_SENTRY_DSN=（同上 or 同一プロジェクトの DSN）
# → pm2 restart senmon（NEXT_PUBLIC_ はビルドに焼かれるため、必要なら再ビルド）
# 確認: Sentry 側にテストイベントが届くこと
```

## 4. nginx 実IP転送（レート制限を正しく効かせる）

`getClientIp` は `x-real-ip` を見る。nginx が実IPを転送していないと、全リクエストがプロキシIP（127.0.0.1 等）に
見え、**レート制限が「全員1キー」**になり一斉出願が逆に詰まる/IP制限が無意味になる。

```nginx
# nginx の location / に以下があること
proxy_set_header X-Real-IP        $remote_addr;
proxy_set_header X-Forwarded-For  $proxy_add_x_forwarded_for;
```

## 5. CSP（コード側は Report-Only で導入済み・将来 enforce）

`next.config.mjs` に `Content-Security-Policy-Report-Only` を本番のみ設定済み。**いまは違反を報告するだけで画面は壊れない**。
本番でしばらく運用し、ブラウザ devtools / Sentry に CSP 違反が出ないことを確認したら、`Content-Security-Policy`（enforce）へ切り替える。
外部に追加で通信する先（フォント CDN、解析等）があれば `connect-src`/`font-src` に追記してから enforce する。

```bash
# .env 編集後は必ず

```
